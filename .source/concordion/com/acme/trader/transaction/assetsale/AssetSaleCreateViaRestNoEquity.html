Create Sell Asset via REST when Equity Not Found

Given

An equity with identifier 669986123 does not exist within the system.

When

A HTTP POST request is made on the following URL 

/trader/transactions

with the message body including a plain text representation of the transaction 

{
  "type" : "Sell Asset",
  "tradeDate" : "2010-08-21",
  "riskClass" : "chill",
  "strategy" : "arrrh",
  "family" : "chazzwazzer",
  "subBook" : "arrrh",
  "coltBook" : "kermit",
  "quantity" : 715,
  "equityIdentifier" : "669986123"
}

with a request Accept header of 

application/json

Then

The HTTP response includes the status 404 (Not Found) and a transaction is not created

The HTTP response should also include in the body 

{"message" : "equity with identifier 669986123 not found"}
