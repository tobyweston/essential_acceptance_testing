Sell LEPO / Buy Asset Booking Tally onto the Price Adjustment in Risk for TRS with no URN

When a LEPO is exercised, a price adjustment cashflow must be recorded. The cashflow represents moneys paid out (from the IB perspective) on buy back of the LEPO and reimbursed via the TRS AP Leg. As there is no position change as a result of the buy back, the cashflow is generated by tallying on and off the instrument using the same quantity but with different prices.

When

Trader receives a request to execute an Sell LEPO / Buy Asset transaction. A request to tally on the TRS AP Leg is made to Risk (via Dragon)

Example

The trade is correctly created and booked but eps returns no URN

Then

A JSON message is sent to Dragon representing the tally onto the instrument (referenced by the instrTic) via a HTTP POST.
