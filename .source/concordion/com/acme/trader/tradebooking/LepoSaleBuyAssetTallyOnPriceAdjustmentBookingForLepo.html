Sell LEPO / Buy Asset Booking Tally On the Price Adjustment in GR for LEPO

When a LEPO is exercised, a price adjustment cashflow must be recorded. The cashflow represents moneys paid out (from the IB perspective) on buy back of the LEPO and reimbursed via the TRS AP Leg. As there is no position change as a result of the buy back, the cashflow is generated by tallying on and off the instrument using the same quantity but with different prices.

When

Trader receives a request to execute an Sell LEPO / Buy Asset transaction. A request to tally off the LEPO is made to GR (via Dragon)

Then

A JSON message is sent to Dragon representing the tally onto the instrument (referenced by the symbol) via a HTTP POST.
