Book IComp in GR

An IComp instrument in GR will reflect the composition of the asset portfolio for a given ETF. It's used to represent the risk associated with the TRS AP leg (as it's the underlying instrument for that leg) although it is not part of the ETF Co-ordination desk's risk view (as they don't actually own the portfolio).

Nightly GR mark runs will value the IComp and this data is used in the subsequent reconciliation process (T+1). Failure to keep the IComp in-sync with the portfolio will result in reconciliation failures and discrepancies in the ETF Co-ordination desk's P&L.

See the IComp Update wiki page for more background.

Given

Today's date is 2011-08-04 and a varied asset portfolio exists

When

Trader makes an IComp booking to reflect the current portfolio composition

Then

A valid JSON message is sent to Dragon via a HTTP POST containing all the portfolio consituents (for a known tradableInfoID representing the IComp instrument)
