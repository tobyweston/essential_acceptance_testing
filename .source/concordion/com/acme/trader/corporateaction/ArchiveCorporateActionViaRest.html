Both of them are not archived

User has the TRADER_ETF_Coordination_Desk role, with permission "Archive Corporate action"

When

The user archives the corporate action with id 2658961 via REST

A HTTP PUT request is made to archive the corporate action resource on the URL 

/trader/corporate_actions
      {
        "id" : "2658961",
        "archived" : true
      }
Note: all fields but "id" and "archived" will be ignored.

Then

The corporate action 2658961 is archived and the HTTP response includes the status 200 (OK).
