Get Equity via REST - Missing, Null and Empty Values from ESM

Given

ESM returns empty fields for an equity with UBS ID 020651202

ESM will return the following JSON

{
    "_ns_": {
      "_": "urn:lingua:instrument/model"
    },
    "EquityInstrumentMsg": {
      "schemaVersion": "1.4.4",
      "event": {
      },
      "issuer": {
        "_ns_": {
          "ns0": "urn:lingua:instrument/types"
        },
        "_type_": "ns0:EqInstrumentIssuer",
        "countryOfIncorporation": "",
        "icbSupersector": ""
      },
      "issue": {
        "restrictedListCode": "RL5",
        "isin": "",
        "bbSecurityType": "",
        "cusip": ""
      },
      "tL": {
        "ubsId": "020651202",
        "currency": "",
        "tradeCountry": "",
        "sedol": "",
        "ric": "",
        "bbTickerExchange": "",
        "tradingLineName": ""
      },
      "settle": {
      }
    }
  }

and returns with Created

When

The list of equities are retrieved from APMan

The equities as a resource is queried with a HTTP GET on the URL... 

/apman/equities

with a "accept" mime-type of... 

application/json

Then

APMan responds as follows:

Example

[
  {
    "id" : "020651202",
    "name" : null,
    "currency" : null,
    "tradeCountry" : null,
    "countryOfIncorporation" : null,
    "ubsId" : "020651202",
    "sedol" : null,
    "ric" : null,
    "bloombergTicker" : null,
    "bloombergSecurityType" : null,
    "isin" : null,
    "cusip" : null,
    "icbSuperSector" : null,
    "region" : null,
    "classification" : null
  }
]

Note - Region, Classification and Country values will only be populated when tradeCountry value is present (using localities.csv)

Cleanup

The equities in the Mock ESM are cleaned up OK
